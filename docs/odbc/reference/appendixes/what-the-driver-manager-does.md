# <a name="what-the-driver-manager-does"></a>Ce que fait le gestionnaire de pilotes
Le tableau suivant résume comment ODBC *3.x* Gestionnaire de pilotes mappe des appels à ODBC *2.x* et ODBC *3.x* pilotes.  
  
|Fonction ou<br /><br /> attribut d'instruction|Commentaires|  
|-----------------------------------------|--------------|  
|SQL_ATTR_FETCH_BOOKMARK_PTR|Pointe vers le signet à utiliser avec **SQLFetchScroll**. Détails d’implémentation sont les suivantes :<br /><br /> -Lorsqu’une application définit cela dans une application ODBC *2.x* pilote ODBC *3.x* Gestionnaire de pilotes met en cache. Il déréférence le pointeur et transmet la valeur à ODBC *2.x* pilote dans le *FetchOffset* argument de **SQLExtendedFetch** lorsque **SQLFetchScroll**  est appelée ultérieurement par l’application.<br />-Lorsqu’une application définit cela dans une application ODBC *3.x* pilote ODBC *3.x* Gestionnaire de pilotes passe l’appel au pilote.|  
|SQL_ATTR_ROW_STATUS_PTR|Pointe vers le tableau d’état des lignes remplies par **SQLFetch**, **SQLFetchScroll**, **SQLBulkOperations**, et **SQLSetPos**. Détails d’implémentation sont les suivantes :<br /><br /> -Lorsqu’une application définit cela dans une application ODBC *2.x* pilote ODBC *3.x* Gestionnaire de pilotes met en cache sa valeur. Cette valeur est transmise à ODBC *2.x* pilote dans le *RowStatusArray* argument de **SQLExtendedFetch** lorsque **SQLFetchScroll** ou **SQLFetch** est appelée.<br />-Lorsqu’une application définit cela dans une application ODBC *3.x* pilote ODBC *3.x* Gestionnaire de pilotes passe l’appel au pilote.<br />-Dans un état S6, si une application définit SQL_ATTR_ROW_STATUS_PTR et appelle ensuite **SQLBulkOperations** (avec un *opération* de SQL_ADD) ou **SQLSetPos** sans appeler d’abord **SQLFetch** ou **SQLFetchScroll**, SQLSTATE HY011 (attribut ne peut pas être défini maintenant) est retournée.|  
|SQL_ATTR_ROWS_FETCHED_PTR|Pointe vers la mémoire tampon dans laquelle **SQLFetch** et **SQLFetchScroll** retourner le nombre de lignes extraites. Détails d’implémentation sont les suivantes :<br /><br /> -Lorsqu’une application définit cela dans une application ODBC *2.x* pilote ODBC *3.x* Gestionnaire de pilotes met en cache sa valeur. Cette valeur est transmise à ODBC *2.x* pilote dans le *RowCountPtr* argument de **SQLExtendedFetch** lorsque **SQLFetch** ou **SQLFetchScroll** est appelée par l’application.<br />-Lorsqu’une application définit cela dans une application ODBC *3.x* pilote ODBC *3.x* Gestionnaire de pilotes passe l’appel au pilote.|  
|SQL_ATTR_ROW_ARRAY_SIZE|Définit la taille de l’ensemble de lignes. Détails d’implémentation sont les suivantes :<br /><br /> -Lorsqu’une application définit cela dans une application ODBC *2.x* pilote ODBC *3.x* Gestionnaire de pilotes est mappé à l’attribut d’instruction SQL_ROWSET_SIZE.<br />-Lorsqu’une application définit cela dans une application ODBC *3.x* pilote ODBC *3.x* Gestionnaire de pilotes passe l’appel au pilote.<br />-Quand une application fonctionne avec une application ODBC *3.x* pilote appelle **SQLSetScrollOptions**, SQL_ROWSET_SIZE est définie à la valeur de la *la RowsetSize* argument si sous-jacent pilote ne prend pas en charge **SQLSetScrollOptions**.|  
|SQL_ROWSET_SIZE|Définit la taille de l’ensemble de lignes utilisée par **SQLExtendedFetch** lorsque **SQLExtendedFetch** est appelée par une application ODBC *2.x* application. Détails d’implémentation sont les suivantes :<br /><br /> -Lorsqu’une application définit, ODBC *3.x* Gestionnaire de pilotes passe l’appel au pilote, quelle que soit la version du pilote.<br />-Quand une application fonctionne avec une application ODBC *2.x* pilote appelle **SQLSetScrollOptions**, SQL_ROWSET_SIZE est définie à la valeur de la **la RowsetSize** argument.|  
|**SQLBulkOperations**|Effectue une insertion opération ou la mise à jour, suppression ou extraction par les opérations de signet. Détails d’implémentation sont les suivantes :<br /><br /> -Lorsqu’une application appelle **SQLBulkOperations** avec un *opération* de SQL_ADD dans une application ODBC *2.x* pilote ODBC *3.x* pilote Manager mappe à **SQLSetPos** avec un *opération* de SQL_ADD.<br />-Lorsque vous travaillez avec une application ODBC *2.x* pilote qui ne prend pas en charge **SQLSetPos** avec un *opération* de SQL_ADD, ODBC *3.x* pilote Le gestionnaire n’est pas mappé **SQLSetPos** avec un *opération* de SQL_ADD à **SQLBulkOperations** avec un *opération* de SQL _AJOUTER. Il s’agit, car **SQLBulkOperations** ne peut pas être appelée dans un état S7, le ODBC dans *2.x* a été le seul état dans lequel **SQLSetPos** peut être appelée.<br />-Si l’application appelle **SQLBulkOperations** avec un *opération* de SQL_ADD dans une application ODBC *2.x* pilote avant d’appeler **SQLFetchScroll**, ODBC *3.x* Gestionnaire de pilotes retourne une erreur.|  
|**SQLExtendedFetch**|Retourne l’ensemble de lignes spécifié. À l’exception de la restriction tout à fait, ODBC *3.x* Gestionnaire de pilotes passe des appels aux **SQLExtendedFetch** au pilote, quelle que soit la version du pilote.|  
|**SQLFetch**|Retourne l’ensemble de lignes suivant. Détails d’implémentation sont les suivantes :<br /><br /> -Lorsqu’une application appelle **SQLFetch** dans une application ODBC *2.x* pilote ODBC *3.x* Gestionnaire de pilotes mappe à **SQLExtendedFetch**. Le *FetchOrientation* argument de **SQLExtendedFetch** est définie sur SQL_FETCH_NEXT. Le Gestionnaire de pilotes utilise la valeur mise en cache de l’attribut d’instruction SQL_ATTR_ROW_STATUS_PTR pour le *RowStatusArray* argument et la valeur mise en cache de l’attribut d’instruction SQL_ATTR_ROWS_FETCHED_PTR pour le  *RowCountPtr* argument.<br />-Une application ODBC *3.x* application pouvez mélanger les appels à **SQLFetch** et **SQLFetchScroll** dans une application ODBC *2.x* pilote, car ODBC *3.x* mappe de gestionnaire de pilotes **SQLFetch** à **SQLExtendedFetch** lorsqu’une application appelle dans une application ODBC *2.x* pilote.<br />-If une ODBC *2.x* pilote ne prend pas en charge **SQLExtendedFetch**, ODBC *3.x* Gestionnaire de pilotes ne mappe pas **SQLFetch** ou **SQLFetchScroll** à **SQLExtendedFetch** lorsqu’une application appelle dans ce pilote. Si l’application tente de définir les SQL_ATTR_ROW_ARRAY_SIZE sur une valeur supérieure à 1, SQLSTATE HYC00 (fonctionnalité facultative non implémentée) est retournée.<br />-Sauf pour les restrictions tout à fait, ODBC *3.x* Gestionnaire de pilotes passe des appels aux **SQLFetch** au pilote, quelle que soit la version du pilote.|  
|**SQLFetchScroll**|Retourne l’ensemble de lignes spécifié. Détails d’implémentation sont les suivantes :<br /><br /> -Lorsqu’une application appelle **SQLFetchScroll** dans une application ODBC *2.x* pilote ODBC *3.x* Gestionnaire de pilotes mappe à **SQLExtendedFetch**. Il utilise la valeur mise en cache de l’attribut d’instruction SQL_ATTR_ROW_STATUS_PTR pour le *RowStatusArray* argument et la valeur mise en cache de l’attribut d’instruction SQL_ATTR_ROWS_FETCHED_PTR pour le *RowCountPtr* argument. Si le *FetchOrientation* argument dans **SQLFetchScroll** est SQL_FETCH_BOOKMARK, il utilise la valeur mise en cache de l’attribut d’instruction SQL_ATTR_FETCH_BOOKMARK_PTR pour la *FetchOffset*  argument et retourne une erreur si le *FetchOffset* argument de **SQLFetchScroll** est pas égal à 0.<br />-Lorsqu’une application appelle cela dans une application ODBC *3.x* pilote ODBC *3.x* Gestionnaire de pilotes passe l’appel au pilote.|  
|**SQLSetPos**|Effectue des opérations positionnées différents. ODBC *3.x* Gestionnaire de pilotes passe des appels aux **SQLSetPos** au pilote, quelle que soit la version du pilote.|  
|**SQLSetScrollOptions**|Lorsque le Gestionnaire de pilotes mappe **SQLSetScrollOptions** pour une application fonctionne avec une application ODBC *3.x* pilote qui ne prend pas en charge **SQLSetScrollOptions**, le pilote Le gestionnaire définit l’option d’instruction SQL_ROWSET_SIZE, pas l’attribut d’instruction SQL_ATTR_ROW_ARRAY_SIZE, à la *la RowsetSize* argument dans **SQLSetScrollOption**. Par conséquent, **SQLSetScrollOptions** ne peut pas être utilisé par une application lors de l’extraction de plusieurs lignes par un appel à **SQLFetch** ou **SQLFetchScroll**. Il peut être uniquement utilisé lorsque l’extraction de plusieurs lignes par un appel à **SQLExtendedFetch**.|
