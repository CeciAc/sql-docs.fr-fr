# <a name="standard-database-access-architectures"></a>Architectures de l’accès aux bases de données standard
Examinez les composants d’accès de base de données décrites dans la section précédente, il s’avère que deux d'entre eux - programmation interfaces et protocoles de flux de données - sont de bons candidats pour la normalisation. Les deux autres composants - mécanisme IPC et protocoles réseau - résident non seulement à un niveau trop bas, mais ils sont fortement dépendants sur le réseau et le système d’exploitation. Il existe également une troisième approche - passerelles - qui offre des possibilités de normalisation.  
  
 Cette section contient les rubriques suivantes.  
  
-   [Interface de programmation standard](../../odbc/reference/standard-programming-interface.md)  
  
-   [Protocole de flux de données standard](../../odbc/reference/standard-data-stream-protocol.md)  
  
-   [Passerelle standard](../../odbc/reference/standard-gateway.md)
