# <a name="understanding-admin-console-alerts-in-analytics-platform-system"></a>Présentation des alertes de la Console d’administration d’Analytique Platform System
Les alertes s’affichent dans l’appliance **Console d’administration** et dans **System Center Operations Manager** (SCOM). Utilisez cette liste d’alertes pour vous aider à identifier les alertes qui nécessitent un examen complémentaire.  
  
Pour plus d’informations sur la connexion à la **Console d’administration** en utilisant Internet Explorer, consultez [surveiller l’Appliance à l’aide de la Console d’administration &#40;Analytique Platform System&#41;](monitor-the-appliance-by-using-the-admin-console.md). Pour plus d’informations sur **SCOM**, consultez [surveiller l’Appliance à l’aide de System Center Operations Manager &#40;Analytique Platform System&#41;](monitor-the-appliance-by-using-system-center-operations-manager.md)  
  
Pour plus d’informations sur l’obtention d’informations sur l’alertes à l’aide de Transact\-SQL, consultez [surveiller l’Appliance par à l’aide des vues système &#40;Analytique Platform System&#41;](monitor-the-appliance-by-using-system-views.md).  
  
## <a name="types-of-alerts"></a>Types d’alertes  
Les noms d’alerte qui indiquent un état NORMAL ne nécessitent pas généralement l’investigation. Les noms d’alerte qui contiennent les mots NON_CRITICAL parfois nécessitent une action. Examen complémentaire est requis pour tous les autres types d’alertes.
